# NeRFly — Claude Code Context

## Project summary

Research demo for a **Next-Best-View (NBV) planning** PhD project.
A UAV autonomously orbits an object in simulation, captures 2D images, and
those images are fed into COLMAP to produce a 3D reconstruction.

**Design principles (carry these into every suggestion):**
- Prioritise correctness and clarity over brevity — this is a research demo,
  not a production system.
- Each phase must be independently runnable and debuggable.
- Prefer explicit, well-commented scripts over terse ones.

forward-facing mono camera.

Launch command (must be running before the ROS 2 stack):
```bash
cd ~/PX4-Autopilot

```

MAVLink UDP port opened by PX4 SITL: `localhost:14540`
MAVROS connects to it at: `udp://:14540@localhost:14557`

/world/default/model/x500_mono_cam_0/link/camera_link/sensor/camera/camera_info

## Mission parameters (`config/mission_params.yaml`)

| Parameter | Default | Meaning |
|---|---|---|
| `orbit_radius` | `5.0` | Circle radius in metres |
| `orbit_height` | `3.0` | Flight altitude AGL in metres |
| `n_images` | `36` | Capture points (36 = every 10°) |
| `target_x` | `0.0` | Orbit centre X (local ENU, metres) |
| `target_y` | `0.0` | Orbit centre Y (local ENU, metres) |
| `waypoint_tolerance` | `0.3` | Arrival threshold in metres |
| `waypoint_dwell` | `0.5` | Hold time at each WP before capture (s) |
| `image_save_dir` | `~/nbv_images` | Output directory for PNG frames |

## Mission state machine (`mission_node.py`)

```
WAIT_FCU → PREARM_STREAM → WAIT_OFFBOARD → WAIT_ARM →
TAKEOFF → ORBIT → RETURN_HOME → WAIT_LAND → DONE
```

- Setpoints are streamed at **20 Hz** at all times (PX4 exits OFFBOARD mode if
  the stream stops for >0.5 s).
- All MAVROS service calls use `call_async` + `add_done_callback`.  The state
  machine polls `current_state` to confirm transitions — never blocks inside a
  callback (avoids single-threaded executor deadlock).
- Yaw at each orbit waypoint faces inward toward the orbit centre.

## Known issues and fixes

### MAVROS crash on launch (`invalid allocator`)

**Symptom:** `mavros_node` dies immediately with:
```
create_subscription() called for existing topic ... with incompatible type CompanionProcessStatus_
invalid allocator
```

**Cause:** In `ros-jazzy-mavros`, the `companion_process_status` plugin
creates a subscription to a topic already registered by `sys_status` with an
incompatible type.  RCL mishandles the error, corrupts its allocator state,
and aborts.

**Fix (already applied in `mission.launch.py`):**
```python
'plugin_blocklist': [
    'companion_process_status',
    'adsb',
    'cellular_status',
    'cam_imu_sync',
]
```

## Build and run (quick reference)

```bash
# 0. Activate conda env (Python packages)
conda activate nerfly

# 1. Start PX4 SITL + Gazebo  (Terminal A — keep running)
cd ~/PX4-Autopilot
make px4_sitl gz_x500_mono_cam

# 2. Source ROS 2  (every new terminal)
source /opt/ros/jazzy/setup.bash

# 3. Build the ROS 2 package  (once, or after any code change)
cd ~/PhD/NeRFly
colcon build --packages-select nbv_demo
source install/setup.bash

# 4. Launch the demo  (Terminal B)
ros2 launch nbv_demo mission.launch.py

# 5. Watch saved images  (Terminal C, optional)
watch -n1 ls ~/nbv_images/
```
